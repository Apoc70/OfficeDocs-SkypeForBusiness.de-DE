<span data-ttu-id="19265-105">Skype for Business Server muss sicher und nahtlos mit anderen Anwendungen und Serverprodukten kommunizieren können.</span><span class="sxs-lookup"><span data-stu-id="19265-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="19265-106">So können Sie beispielsweise Skype for Business Server so konfigurieren, dass Kontaktdaten und/oder Archivierungsdaten in Microsoft Exchange Server 2013 gespeichert werden. Dies kann jedoch nur erfolgen, wenn Skype for Business Server und Exchange sicher miteinander kommunizieren können.</span><span class="sxs-lookup"><span data-stu-id="19265-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="19265-107">Ebenso können Sie eine Skype for Business Server-Konferenz in Office Web Apps Server planen. auch dies kann nur erfolgen, wenn sich die beiden Server (SharePoint und Skype for Business Server) einander vertrauen.</span><span class="sxs-lookup"><span data-stu-id="19265-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="19265-108">Obwohl es möglich ist, einen Authentifizierungsmechanismus für die Kommunikation zwischen Skype for Business Server und Exchange, aber einen separaten Mechanismus für die Kommunikation zwischen Skype for Business Server und SharePoint zu verwenden, besteht ein besserer und effizienterer Ansatz in der Verwendung einer standardisierte Methode für alle Server-zu-Server-Authentifizierung und-Autorisierung.</span><span class="sxs-lookup"><span data-stu-id="19265-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="19265-109">Die Verwendung einer einzelnen standardisierten Methode für die Server-zu-Server-Authentifizierung ist der Ansatz von Skype for Business Server.</span><span class="sxs-lookup"><span data-stu-id="19265-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="19265-110">Mit der Office Server 2013-Version von Skype for Business Server (sowie anderen Microsoft-Serverprodukten, einschließlich Exchange Server und SharePoint Server), wurde das OAuth (Open Authorization)-Protokoll für die Server-zu-Server-Authentifizierung unterstützt und Autorisierungs.</span><span class="sxs-lookup"><span data-stu-id="19265-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="19265-111">Bei OAuth wird ein Standard Autorisierungs Protokoll, das von einer Reihe von Hauptwebsites, Benutzeranmeldeinformationen und Kennwörtern verwendet wird, nicht von einem Computer an einen anderen weitergegeben.</span><span class="sxs-lookup"><span data-stu-id="19265-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="19265-112">Stattdessen basiert Authentifizierung und Autorisierung auf dem Austausch von Sicherheitstokens; Diese Token gewähren für einen bestimmten Zeitraum Zugriff auf eine bestimmte Gruppe von Ressourcen.</span><span class="sxs-lookup"><span data-stu-id="19265-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="19265-113">Die OAuth-Authentifizierung umfasst in der Regel drei Parteien: einen einzelnen autorisierungsserver und die beiden Realms, die miteinander kommunizieren müssen.</span><span class="sxs-lookup"><span data-stu-id="19265-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="19265-114">(Sie können auch eine Server-zu-Server-Authentifizierung ausführen, ohne einen autorisierungsserver zu verwenden, ein Prozess, der später in diesem Dokument erläutert wird.) Sicherheitstoken werden von dem autorisierungsserver (auch als Sicherheitstokenserver bezeichnet) für die beiden Bereiche ausgestellt, die kommuniziert werden müssen. Diese Token stellen sicher, dass die Kommunikation, die von einem Bereich stammt, vom anderen Bereich als vertrauenswürdig eingestuft wird.</span><span class="sxs-lookup"><span data-stu-id="19265-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) Security tokens are issued by the authorization server (also known as a security token server) to the two realms that need to communicate; these tokens verify that communications originating from one realm should be trusted by the other realm.</span></span> <span data-ttu-id="19265-115">Beispielsweise kann der autorisierungsserver Token ausgeben, die sicherstellen, dass Benutzer aus einem bestimmten Skype for Business-Serverbereich auf einen bestimmten Exchange-Realm zugreifen können, und umgekehrt.</span><span class="sxs-lookup"><span data-stu-id="19265-115">For example, the authorization server might issue tokens that verify that users from a specific Skype for Business Server realm are able to access a specified Exchange realm, and vice-versa.</span></span>
  
> [!NOTE]
> <span data-ttu-id="19265-116">Ein Bereich ist einfach ein Sicherheitscontainer.</span><span class="sxs-lookup"><span data-stu-id="19265-116">A realm is simply a security container.</span></span> <span data-ttu-id="19265-117">Standardmäßig verwendet Skype for Business Server Ihre SIP-Standarddomäne als OAuth-Realm.</span><span class="sxs-lookup"><span data-stu-id="19265-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="19265-118">Weitere SIP-Namespaces werden der Liste mit alternativen Antragstellernamen im OAuth-Zertifikat hinzugefügt.</span><span class="sxs-lookup"><span data-stu-id="19265-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="19265-119">Skype for Business Server unterstützt drei Szenarien für die Server-zu-Server-Authentifizierung.</span><span class="sxs-lookup"><span data-stu-id="19265-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="19265-120">Mit Skype for Business Server haben Sie folgende Möglichkeiten:</span><span class="sxs-lookup"><span data-stu-id="19265-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="19265-121">Konfigurieren Sie die Server-zu-Server-Authentifizierung zwischen einer lokalen Installation von Skype for Business Server und einer lokalen Installation von Exchange und/oder SharePoint Server.</span><span class="sxs-lookup"><span data-stu-id="19265-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="19265-122">Konfigurieren Sie die Server-zu-Server-Authentifizierung zwischen zwei Office 365-Komponenten (beispielsweise zwischen Microsoft Exchange Server und Skype for Business Server oder zwischen Skype for Business Server und SharePoint).</span><span class="sxs-lookup"><span data-stu-id="19265-122">Configure server-to-server authentication between a pair of Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="19265-123">Konfigurieren Sie die Server-zu-Server-Authentifizierung in einer standortübergreifenden Umgebung (Server-zu-Server-Authentifizierung zwischen einem lokalen Server und einer Office 365-Komponente).</span><span class="sxs-lookup"><span data-stu-id="19265-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and an Office 365 component).</span></span>
    
<span data-ttu-id="19265-124">Beachten Sie, dass zu diesem Zeitpunkt nur Exchange 2013, SharePoint Server, lync Server 2013, Skype for Business Server 2015 und Skype for Business 2019 die Server-zu-Server-Authentifizierung unterstützen. Wenn Sie einen dieser Server nicht ausführen, können Sie die OAuth-Authentifizierung nicht vollständig implementieren.</span><span class="sxs-lookup"><span data-stu-id="19265-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="19265-125">Es sollte auch darauf hingewiesen werden, dass die Server-zu-Server-Authentifizierung optional ist: Wenn Skype for Business Server nicht mit anderen Servern (wie Exchange) kommunizieren muss, kann die Server-zu-Server-Authentifizierung komplett übersprungen werden.</span><span class="sxs-lookup"><span data-stu-id="19265-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="19265-126">Wenn die Server-zu-Server-Authentifizierung bereits für lync Server 2013 und andere Anwendungen konfiguriert ist, müssen Sie Sie nicht für Skype for Business Server erneut ausführen.</span><span class="sxs-lookup"><span data-stu-id="19265-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="19265-127">Die Server-zu-Server-Authentifizierung ist jedoch erforderlich, wenn Sie einige der Features in Skype for Business Server verwenden möchten, beispielsweise den "Unified Contact Store".</span><span class="sxs-lookup"><span data-stu-id="19265-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="19265-128">Mit dem Unified Contact Store werden die Skype for Business Server-Kontaktinformationen in Exchange anstatt in Skype for Business Server gespeichert. auf diese Weise können Benutzer über einen einzigen Satz Kontakte verfügen, auf die in Skype for Business, Outlook oder Outlook Web Access problemlos zugegriffen werden kann.</span><span class="sxs-lookup"><span data-stu-id="19265-128">With unified contact store, Skype for Business Server contact information is stored in Exchange instead of in Skype for Business Server; <span data-ttu-id="19265-131">Damit die Office 365-Version von Skype for Business Server mit Ihrem Exchange-Pendant kommuniziert, muss Skype for Business Server zunächst ein Sicherheitstoken vom autorisierungsserver abrufen.</span><span class="sxs-lookup"><span data-stu-id="19265-131">For the Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="19265-132">Skype for Business Server verwendet dann dieses Sicherheitstoken, um sich selbst für Exchange zu identifizieren.</span><span class="sxs-lookup"><span data-stu-id="19265-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="19265-133">Die Office 365-Version von Exchange muss denselben Vorgang durchlaufen, um mit Skype for Business Server kommunizieren zu können.</span><span class="sxs-lookup"><span data-stu-id="19265-133">The Office 365 version of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="19265-134">Für eine lokale Server-zu-Server-Authentifizierung zwischen zwei Microsoft-Servern muss kein Drittanbieter-Tokenserver verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="19265-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="19265-135">Serverprodukte wie Skype for Business Server und Exchange verfügen über einen integrierten tokenserver, der für Authentifizierungszwecke mit anderen Microsoft-Servern (wie SharePoint Server) verwendet werden kann, die die Server-zu-Server-Authentifizierung unterstützen.</span><span class="sxs-lookup"><span data-stu-id="19265-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="19265-136">Beispielsweise kann Skype for Business Server selbst ein Sicherheitstoken ausgeben und Signieren und dann dieses Token für die Kommunikation mit Exchange verwenden.</span><span class="sxs-lookup"><span data-stu-id="19265-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="19265-137">In einem solchen Fall ist kein Drittanbieter-Tokenserver erforderlich.</span><span class="sxs-lookup"><span data-stu-id="19265-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="19265-138">Um die Server-zu-Server-Authentifizierung für eine lokale Implementierung von Skype for Business Server zu konfigurieren, müssen Sie zwei Schritte ausführen:</span><span class="sxs-lookup"><span data-stu-id="19265-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
  
- <span data-ttu-id="19265-139">Weisen Sie dem integrierten Skype for Business Server-Token-Aussteller ein Zertifikat zu.</span><span class="sxs-lookup"><span data-stu-id="19265-139">Assign a certificate to the built-in Skype for Business Server token issuer.</span></span>
    
- <span data-ttu-id="19265-140">Konfigurieren Sie den Server, mit dem Skype for Business Server kommunizieren soll, als "Partneranwendung".</span><span class="sxs-lookup"><span data-stu-id="19265-140">Configure the server that Skype for Business Server will communicate with to be a "partner application."</span></span> <span data-ttu-id="19265-141">Wenn Skype for Business Server beispielsweise mit Exchange kommunizieren muss, müssen Sie Exchange so konfigurieren, dass es sich um eine Partneranwendung handelt.</span><span class="sxs-lookup"><span data-stu-id="19265-141">For example, if Skype for Business Server needs to communicate with Exchange, you will need to configure Exchange to be a partner application.</span></span>
