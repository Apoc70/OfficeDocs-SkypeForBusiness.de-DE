# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a><span data-ttu-id="74d02-103">Verwalten der Server-zu-Server-Authentifizierung (OAuth) und der Partneranwendungen in Skype for Business Server</span><span class="sxs-lookup"><span data-stu-id="74d02-103">Manage server-to-server authentication (OAuth) and partner applications in Skype for Business Server</span></span>

<span data-ttu-id="74d02-105">Skype for Business Server müssen in der Lage sein, sicher und nahtlos mit anderen Anwendungen und Server Produkten zu kommunizieren.</span><span class="sxs-lookup"><span data-stu-id="74d02-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="74d02-106">Sie können beispielsweise Skype for Business Server so konfigurieren, dass Kontaktdaten und/oder Archivierungsdaten in Microsoft Exchange Server 2013 gespeichert werden. Dies kann jedoch nur erfolgen, wenn Skype for Business Server und Exchange sicher miteinander kommunizieren können.</span><span class="sxs-lookup"><span data-stu-id="74d02-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="74d02-107">Ebenso können Sie eine Skype for Business Server Konferenz in Office-webapps Server planen; Dies kann wiederum nur geschehen, wenn die beiden Server (SharePoint und Skype for Business Server) einander vertrauen.</span><span class="sxs-lookup"><span data-stu-id="74d02-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="74d02-108">Obwohl es möglich ist, einen Authentifizierungsmechanismus für die Kommunikation zwischen Skype for Business Server und Exchange, aber einen separaten Mechanismus für die Skype for Business Server-und SharePoint-Kommunikation zu verwenden, besteht ein besserer und effizienterer Ansatz darin, eine standardisierte Methode für alle Server-zu-Server-Authentifizierung und-Autorisierung zu verwenden.</span><span class="sxs-lookup"><span data-stu-id="74d02-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="74d02-109">Die Verwendung einer einzigen standardisierten Methode für die Server-zu-Server-Authentifizierung ist der Ansatz, der von Skype for Business Server getroffen wird.</span><span class="sxs-lookup"><span data-stu-id="74d02-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="74d02-110">Mit Office Server 2013 Release, Skype for Business Server (sowie anderen Microsoft-Serverprodukten, einschließlich Exchange Server und SharePoint Server), wurde das OAuth (Open Authorization)-Protokoll für die Server-zu-Server-Authentifizierung und-Autorisierung unterstützt.</span><span class="sxs-lookup"><span data-stu-id="74d02-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="74d02-111">Mit OAuth wird ein Standardmäßiges Autorisierungs Protokoll, das von einer Reihe von Hauptwebsites verwendet wird, Benutzeranmeldeinformationen und Kennwörter nicht von einem Computer an einen anderen übergeben.</span><span class="sxs-lookup"><span data-stu-id="74d02-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="74d02-112">Stattdessen basieren Authentifizierung und Autorisierung auf dem Austausch von Sicherheitstoken; Diese Token gewähren Zugriff auf einen bestimmten Satz von Ressourcen für einen bestimmten Zeitraum.</span><span class="sxs-lookup"><span data-stu-id="74d02-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="74d02-113">OAuth-Authentifizierung umfasst normalerweise drei Komponenten: einen Server mit Einzelanmeldung und zwei Bereiche, die miteinander kommunizieren müssen.</span><span class="sxs-lookup"><span data-stu-id="74d02-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="74d02-114">(Sie können auch die Server-zu-Server-Authentifizierung ohne einen autorisierungsserver durchführen, ein Prozess, der weiter unten in diesem Dokument behandelt wird.) Sicherheitstoken werden von dem autorisierungsserver (auch als Sicherheitstokenserver bezeichnet) für die beiden Bereiche ausgestellt, die miteinander kommunizieren müssen. Diese Token stellen sicher, dass die Kommunikation, die von einem Bereich stammt, vom anderen Bereich als vertrauenswürdig eingestuft wird.</span><span class="sxs-lookup"><span data-stu-id="74d02-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) Security tokens are issued by the authorization server (also known as a security token server) to the two realms that need to communicate; these tokens verify that communications originating from one realm should be trusted by the other realm.</span></span> <span data-ttu-id="74d02-115">Beispielsweise kann der autorisierungsserver Token ausgeben, die sicherstellen, dass Benutzer von einem bestimmten Skype for Business Server Bereichs auf einen bestimmten Exchange-Bereich zugreifen können, und umgekehrt.</span><span class="sxs-lookup"><span data-stu-id="74d02-115">For example, the authorization server might issue tokens that verify that users from a specific Skype for Business Server realm are able to access a specified Exchange realm, and vice-versa.</span></span>
  
> [!NOTE]
> <span data-ttu-id="74d02-116">Ein Bereich ist einfach ein Sicherheitscontainer.</span><span class="sxs-lookup"><span data-stu-id="74d02-116">A realm is simply a security container.</span></span> <span data-ttu-id="74d02-117">Standardmäßig verwendet Skype for Business Server Ihre Standard-SIP-Domäne als OAuth-Bereich.</span><span class="sxs-lookup"><span data-stu-id="74d02-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="74d02-118">Zusätzliche SIP-Namespaces werden der Liste alternativer Antragsteller Name im OAuth-Zertifikat hinzugefügt.</span><span class="sxs-lookup"><span data-stu-id="74d02-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="74d02-119">Skype for Business Server unterstützt drei Szenarien für die Server-zu-Server-Authentifizierung.</span><span class="sxs-lookup"><span data-stu-id="74d02-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="74d02-120">Mit Skype for Business Server haben Sie folgende Möglichkeiten:</span><span class="sxs-lookup"><span data-stu-id="74d02-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="74d02-121">Konfigurieren der Server-zu-Server-Authentifizierung zwischen einer lokalen Installation von Skype for Business Server und einer lokalen Installation von Exchange und/oder SharePoint Server.</span><span class="sxs-lookup"><span data-stu-id="74d02-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="74d02-122">Konfigurieren Sie die Server-zu-Server-Authentifizierung zwischen einem Paar von Microsoft 365-oder Office 365-Komponenten (beispielsweise zwischen Exchange Server und Skype for Business Server oder zwischen Skype for Business Server und SharePoint).</span><span class="sxs-lookup"><span data-stu-id="74d02-122">Configure server-to-server authentication between a pair of Microsoft 365 or Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="74d02-123">Konfigurieren Sie die Server-zu-Server-Authentifizierung in einer standortübergreifenden Umgebung (Server-zu-Server-Authentifizierung zwischen einem lokalen Server und einer Microsoft 365-oder Office 365-Komponente).</span><span class="sxs-lookup"><span data-stu-id="74d02-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and a Microsoft 365 or Office 365 component).</span></span>
    
<span data-ttu-id="74d02-124">Beachten Sie, dass zu diesem Zeitpunkt nur Exchange 2013, SharePoint Server, lync Server 2013, Skype for Business Server 2015 und Skype for Business 2019 die Server-zu-Server-Authentifizierung unterstützen. Wenn Sie einen dieser Server nicht ausführen, können Sie die OAuth-Authentifizierung nicht vollständig implementieren.</span><span class="sxs-lookup"><span data-stu-id="74d02-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="74d02-125">Es sollte auch darauf hingewiesen werden, dass die Server-zu-Server-Authentifizierung optional ist: Wenn Skype for Business Server nicht mit anderen Servern (wie Exchange) kommunizieren muss, kann die Server-zu-Server-Authentifizierung insgesamt übersprungen werden.</span><span class="sxs-lookup"><span data-stu-id="74d02-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="74d02-126">Wenn die Server-zu-Server-Authentifizierung bereits für lync Server 2013 und andere Anwendungen konfiguriert ist, müssen Sie Sie nicht für Skype for Business Server erneut ausführen.</span><span class="sxs-lookup"><span data-stu-id="74d02-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="74d02-127">Die Server-zu-Server-Authentifizierung ist jedoch erforderlich, wenn Sie einige der Features in Skype for Business Server verwenden möchten, beispielsweise den "einheitlichen Kontaktspeicher".</span><span class="sxs-lookup"><span data-stu-id="74d02-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="74d02-128">Mit dem einheitlichen Kontaktspeicher werden Skype for Business Server Kontaktinformationen in Exchange statt in Skype for Business Server gespeichert; auf diese Weise können Benutzer eine einzelne Gruppe von Kontakten haben, auf die in Skype for Business, Outlook oder Outlook Web Access leicht zugegriffen werden kann.</span><span class="sxs-lookup"><span data-stu-id="74d02-128">With unified contact store, Skype for Business Server contact information is stored in Exchange instead of in Skype for Business Server; this enables users to have a single set of contacts that is readily accessible from within Skype for Business, Outlook, or Outlook Web Access.</span></span> <span data-ttu-id="74d02-129">Da für den einheitlichen Kontaktspeicher Skype for Business Server zum Freigeben von Informationen für Exchange erforderlich ist, müssen Sie die Server-zu-Server-Authentifizierung verwenden, um das Feature bereitzustellen.</span><span class="sxs-lookup"><span data-stu-id="74d02-129">Because the unified contact store requires Skype for Business Server to share information with Exchange, you must use server-to-server authentication in order to deploy the feature.</span></span> <span data-ttu-id="74d02-130">Die Server-zu-Server-Authentifizierung ist auch erforderlich, wenn Sie die Exchange-Archivierung verwenden, bei der die Protokolle von Chatsitzungen als Exchange-e-Mails statt als einzelne Datenbankeinträge gespeichert werden.</span><span class="sxs-lookup"><span data-stu-id="74d02-130">Server-to-server authentication is also required if you choose to use Exchange archiving, in which the transcripts of instant messaging sessions are saved as Exchange emails rather than as individual database records.</span></span>
  
<span data-ttu-id="74d02-131">Damit die Microsoft 365-oder Office 365-Version von Skype for Business Server mit Ihrem Exchange-Pendant kommuniziert, muss Skype for Business Server zunächst ein Sicherheitstoken vom Autorisierungs Server abrufen.</span><span class="sxs-lookup"><span data-stu-id="74d02-131">For the Microsoft 365 or Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="74d02-132">Skype for Business Server verwendet dann dieses Sicherheitstoken, um sich selbst zu Exchange zu identifizieren.</span><span class="sxs-lookup"><span data-stu-id="74d02-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="74d02-133">Die Microsoft 365-oder Office 365-Versionen von Exchange müssen den gleichen Prozess durchlaufen, um mit Skype for Business Server zu kommunizieren.</span><span class="sxs-lookup"><span data-stu-id="74d02-133">The Microsoft 365 or Office 365 versions of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="74d02-134">Für eine lokale Server-zu-Server-Authentifizierung zwischen zwei Microsoft-Servern muss kein Drittanbieter-Tokenserver verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="74d02-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="74d02-135">Serverprodukte wie Skype for Business Server und Exchange verfügen über einen integrierten tokenserver, der für Authentifizierungszwecke mit anderen Microsoft-Servern (beispielsweise SharePoint Server) verwendet werden kann, die die Server-zu-Server-Authentifizierung unterstützen.</span><span class="sxs-lookup"><span data-stu-id="74d02-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="74d02-136">Beispielsweise können Skype for Business Server ein Sicherheitstoken selbst ausgeben und signieren, dann verwenden Sie dieses Token für die Kommunikation mit Exchange.</span><span class="sxs-lookup"><span data-stu-id="74d02-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="74d02-137">In einem solchen Fall ist ein Drittanbieter-Tokenserver nicht erforderlich.</span><span class="sxs-lookup"><span data-stu-id="74d02-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="74d02-138">Um die Server-zu-Server-Authentifizierung für eine lokale Implementierung von Skype for Business Server konfigurieren zu können, müssen Sie zwei Schritte ausführen:</span><span class="sxs-lookup"><span data-stu-id="74d02-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
  
- <span data-ttu-id="74d02-139">Weisen Sie dem integrierten Skype for Business Server Token-Aussteller ein Zertifikat zu.</span><span class="sxs-lookup"><span data-stu-id="74d02-139">Assign a certificate to the built-in Skype for Business Server token issuer.</span></span>
    
- <span data-ttu-id="74d02-140">Konfigurieren Sie den Server, mit dem Skype for Business Server kommunizieren wird, als "Partneranwendung".</span><span class="sxs-lookup"><span data-stu-id="74d02-140">Configure the server that Skype for Business Server will communicate with to be a "partner application."</span></span> <span data-ttu-id="74d02-141">Wenn Skype for Business Server beispielsweise mit Exchange kommunizieren muss, müssen Sie Exchange als Partneranwendung konfigurieren.</span><span class="sxs-lookup"><span data-stu-id="74d02-141">For example, if Skype for Business Server needs to communicate with Exchange, you will need to configure Exchange to be a partner application.</span></span>
    
> [!NOTE]
> <span data-ttu-id="74d02-142">Bei einer "Partneranwendung" handelt es sich um eine beliebige Anwendung, mit der Skype for Business Server Sicherheitstoken direkt austauschen kann, ohne einen Sicherheits tokenserver eines Drittanbieters durchlaufen zu müssen.</span><span class="sxs-lookup"><span data-stu-id="74d02-142">A "partner application" is any application that Skype for Business Server can directly exchange security tokens with, without having to go through a third-party security token server.</span></span>
