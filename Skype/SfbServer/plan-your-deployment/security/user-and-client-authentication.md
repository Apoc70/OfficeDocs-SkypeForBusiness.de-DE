- <span data-ttu-id="e4338-113">**NTLM-Protokoll** für Benutzer mit Active Directory-Anmeldeinformationen, die über ein Endgerät von außerhalb der Unternehmensfirewall eine Verbindung herstellen.</span><span class="sxs-lookup"><span data-stu-id="e4338-113">**NTLM protocol** for users with Active Directory credentials who are connecting from an endpoint outside the corporate firewall.</span></span> <span data-ttu-id="e4338-114">Der Zugriffs-edgedienst übergibt Anfragen zur Anmeldung an einen Director, falls vorhanden, oder ein Front-End-Server für die Authentifizierung.</span><span class="sxs-lookup"><span data-stu-id="e4338-114">The Access Edge service passes logon requests to a Director, if present, or a Front End Server for authentication.</span></span> <span data-ttu-id="e4338-115">Der Zugriffs-edgedienst selbst führt keine Authentifizierung.</span><span class="sxs-lookup"><span data-stu-id="e4338-115">The Access Edge service itself performs no authentication.</span></span>

- <span data-ttu-id="e4338-p107">Das **Digestprotokoll** für sogenannte anonyme Benutzer. <span data-ttu-id="e4338-121">Skype für Business Server-Authentifizierung besteht aus zwei Phasen:</span><span class="sxs-lookup"><span data-stu-id="e4338-121">Skype for Business Server authentication consists of two phases:</span></span>

<span data-ttu-id="e4338-p109">Die Benutzervertrauenswürdigkeit ist mit jeder Nachricht verbunden, die von einem Benutzer stammt, nicht mit der Benutzeridentität selbst. <span data-ttu-id="e4338-128">Benutzer mit gültigen Anmeldeinformationen, die von einem Verbundpartner ausgegeben wurden, sind vertrauenswürdig, erhalten aber optional aufgrund zusätzlicher Einschränkungen nicht sämtliche Berechtigungen, die internen Benutzern erteilt werden.</span><span class="sxs-lookup"><span data-stu-id="e4338-128">Users with valid credentials issued by a federated partner are trusted but optionally prevented by additional constraints from enjoying the full range of privileges accorded to internal users.</span></span>
