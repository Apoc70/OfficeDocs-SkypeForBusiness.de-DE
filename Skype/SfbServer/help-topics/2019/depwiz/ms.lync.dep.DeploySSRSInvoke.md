# <a name="sql-server-reporting-services-invoke"></a>SQL Server Reporting Services (Aufruf)
 
Nach der Bereitstellung der erforderlichen Informationen für die Bereitstellung von Monitoring Server-Berichte in der Microsoft SQL Server-Bericht Services, zeigt die Seite Befehle ausführen eine Zusammenfassung der Befehle, die ausgegeben werden, um die Berichte zu SQL Server Reporting installieren Dienste.
  
Überprüfen Sie die Befehlszusammenfassung und notieren Sie etwaige angezeigte Fehler- oder Warnmeldungen. Wenn eine Protokolldatei erstellt wurde, wählen Sie diese unter der Zusammenfassung in der Dropdownliste aus und klicken Sie auf **Protokoll anzeigen**, um die Protokolldatei anzuzeigen.
  
> [!IMPORTANT]
> Für die Reporting Services-Berichte zur erfolgreichen Bereitstellung, und klicken Sie auf die Berichte zuzugreifen, nachdem die Bereitstellung abgeschlossen ist, müssen Sie TCP/IP-Port 80 (und optional, TCP-Port 443 für SSL, wenn Sie die Reporting Services ein Zertifikat zuweisen) öffnen Sie in der Windows-Firewall mit erweiterter Sicherheit auf dem SqlServer. Weitere Informationen hierzu finden Sie unter [Konfigurieren der Windows-Firewall für SQL Server-Zugriff](https://go.microsoft.com/fwlink/p/?linkId=218031) für Microsoft SQL Server 2008 R2.
  
Klicken Sie nach der Überprüfung der Zusammenfassung auf **Fertig stellen** , um die Installation der Berichte für die SQL Server Reporting Services abzuschließen.
