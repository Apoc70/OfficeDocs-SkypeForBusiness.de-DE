# <a name="legacy-merge-next-hop"></a>Legacyzusammenführung – Nächster Hop
 
Der **FQDN des nächsten Hop** identifiziert den nächsten Hop-Server, auf den der Legacy-Edgeserver den internen Datenverkehr leitet. Geben Sie den vollqualifizierten Domänennamen (Fully Qualified Domain Name, FQDN) des Next Hop-Servers ein, auf den dieser Edgeserver den internen Datenverkehr leitet, und wählen Sie dann aus, ob es sich um einen **Front-End-Pool** oder einen **Director**handelt
