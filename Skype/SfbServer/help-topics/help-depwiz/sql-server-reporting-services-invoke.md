# <a name="sql-server-reporting-services-invoke"></a>SQL Server Reporting Services (Aufruf)
 
Nachdem die erforderlichen Informationen für die Bereitstellung der Monitoring Server-Berichte an Microsoft SQL Server 2008 R2 oder an Microsoft SQL Server 2012-Berichtsdienste bereitgestellt wurden, zeigt die Seite Execute-Befehle eine Zusammenfassung der Befehle an, die zum Installieren des Berichte an SQL Server Reporting Services.
  
Überprüfen Sie die Befehlszusammenfassung und notieren Sie etwaige angezeigte Fehler- oder Warnmeldungen. Wenn eine Protokolldatei erstellt wurde, wählen Sie diese unter der Zusammenfassung in der Dropdownliste aus und klicken Sie auf **Protokoll anzeigen**, um die Protokolldatei anzuzeigen.
  
> [!IMPORTANT]
> Damit die Reporting Services-Berichte erfolgreich bereitgestellt werden können und nach Abschluss der Bereitstellung auf die Berichte zugegriffen werden kann, muss TCP/IP-Port 80 (und optional TCP-Port 443 für SSL vorhanden sein, wenn Sie dem Reporting Services ein Zertifikat zuweisen), das in der Windows-Firewall mit erweiterter Sicherheit auf dem SQL Server geöffnet ist. Ausführliche Informationen finden Sie unter [Konfigurieren der Windows-Firewall, damit SQL Server Access](https://go.microsoft.com/fwlink/p/?linkId=218031) für Microsoft SQL Server 2008 R2 zugelassen wird.
  
Nachdem Sie die Zusammenfassung überprüft haben, klicken Sie auf **Fertig stellen** , um die Installation der Berichte für SQL Server Reporting Services abzuschließen.
