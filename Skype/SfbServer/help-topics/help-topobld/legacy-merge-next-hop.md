# <a name="legacy-merge-next-hop"></a>Legacyzusammenführung – Nächster Hop
 
Der **Nächste Hop FQDN** identifiziert den nächsten Hopserver auf dem Edgeserver der Vorversion internen Datenverkehr weitergeleitet. Geben Sie den vollqualifizierten Domänennamen (FQDN) des nächsten Hopservers, dem dieser Edge-Server den internen Datenverkehr weiterleiten, und wählen Sie dann aus, ob es sich um einen **Front-End-Pool** oder einen **Director**ist.
