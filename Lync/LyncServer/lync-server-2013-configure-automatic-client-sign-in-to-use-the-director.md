# <a name="configure-automatic-client-sign-in-to-use-the-director-in-lync-server-2013"></a>Konfigurieren der automatischen Client Sign-In für die Verwendung des Directors in lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Letztes Änderungsstand des Themas:** 2012-09-08_

Wenn Sie eine lync Server 2013, einen Director oder einen Director-Pool bereitstellen, wird empfohlen, die automatische Client Sign-In als bewährte Methode zu verwenden. Ausführliche Informationen zum Konfigurieren von DNS-Servern für die automatische Clientanmeldung finden Sie unter [DNS Requirements for Automatic Client Sign-in in lync Server 2013](lync-server-2013-dns-requirements-for-automatic-client-sign-in.md) in der Planungsdokumentation.

Wenn Sie die automatische Clientanmeldung bereits bereitgestellt haben, finden Sie in den folgenden Abschnitten Informationen zur Konfiguration dieser Funktion für Ihre Director-Server.

<div>

## <a name="single-director-instance"></a>Einzelne Director-Instanz

Wenn Sie die automatische Client Sign-In bereits bereitgestellt haben und auf ein Front-End-Server oder ein Front-End-Pool zeigt, müssen Sie den DNS-SRV-Eintrag so ändern, dass er auf den Director verweist.

</div>

<div>

## <a name="director-pool"></a>Director-Pool

Wenn Sie die automatische Client Sign-In bereits bereitgestellt haben und auf ein Front-End-Server oder ein Front-End-Pool zeigt, müssen Sie den DNS-SRV-Eintrag so ändern, dass er auf den Directorpool verweist.

</div>
