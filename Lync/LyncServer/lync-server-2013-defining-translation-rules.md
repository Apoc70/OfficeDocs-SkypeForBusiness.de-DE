_**Letztes Änderungsstand des Themas:** 2013-02-22_

Lync Server 2013 Enterprise-VoIP leitet Anrufe basierend auf Telefonnummern normalisiert in das E. 164-Format weiter. Dies bedeutet, dass alle gewählten Zeichenfolgen im E. 164-Format zum Zweck der Durchführung von Reverse Number Lookup (können) normalisiert werden müssen, damit Sie in den entsprechenden SIP-URI übersetzt werden können. Lync Server 2013 bietet die Möglichkeit, die aufgerufene ID und die Anrufer-ID-Präsentation zu bearbeiten.

In diesem Abschnitt wird erläutert, wie die angerufene ID und die Anrufer-ID manipuliert werden.
