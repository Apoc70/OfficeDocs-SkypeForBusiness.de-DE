# <a name="configuring-lync-server-2013-to-route-to-a-sipcsta-gateway"></a>Konfigurieren lync Server 2013 für die Weiterleitung an ein SIP/CSTA-Gateway

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Letztes Änderungsstand des Themas:** 2012-10-05_

Ein SIP/CSTA-Gateway ist ein Gateway zwischen SIP und einer computergestützten Telekommunikationsanwendung (Computer-Supported Telecommunications Application, CSTA). Ein SIP/CSTA-Gateway stellt die Schnittstelle zwischen einer vorhandenen PBX-Anlage (Private Branch Exchange) und lync Server zum Weiterleiten von Remote Anruf Steuerungsanforderungen an die Nebenstellenanlage bereit. Nachdem Sie ein SIP/CSTA-Gateway installiert haben, müssen Sie für jeden lync Server Pool, für den Sie die Remoteanrufsteuerung konfigurieren möchten, die folgenden Verfahren ausführen:

  - [Konfigurieren einer statischen Route für die Remoteanrufsteuerung in lync Server 2013](lync-server-2013-configure-a-static-route-for-remote-call-control.md)

  - [Konfigurieren eines Eintrags einer vertrauenswürdigen Anwendung für die Remoteanrufsteuerung in lync Server 2013](lync-server-2013-configure-a-trusted-application-entry-for-remote-call-control.md)
