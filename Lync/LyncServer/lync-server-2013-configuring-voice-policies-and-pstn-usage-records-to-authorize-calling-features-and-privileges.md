<span data-ttu-id="40b64-104">Eine *VoIP-Richtlinie* ermöglicht eine Reihe von Anruffunktionen und ordnet einen oder mehrere PSTN-Verwendungsdaten Sätze an, um die Anruffeatures und Berechtigungen von Benutzern zu definieren, denen die Richtlinie zugewiesen ist.</span><span class="sxs-lookup"><span data-stu-id="40b64-104">A *voice policy* enables a set of calling features and associates one or more PSTN usage records to define the calling features and permissions of users who are assigned the policy.</span></span>

<span data-ttu-id="40b64-105">Der VoIP-Richtlinienbereich kann entweder ein *Standort* sein (der die Standardfeatures und-Berechtigungen für eine Netzwerk Website definiert) oder ein *Benutzer* (der die Features und Berechtigungen definiert, die für einzelne Benutzer oder Gruppen zugewiesen werden sollen).</span><span class="sxs-lookup"><span data-stu-id="40b64-105">Voice policy scope can be either *Site* (which defines the default features and permissions for a network site) or *User* (which defines the features and permissions to be assigned on a per-user or group basis).</span></span> <span data-ttu-id="40b64-106">Benutzer, denen keine VoIP-Richtlinie zugewiesen ist, werden automatisch der globalen Richtlinie zugewiesen, bei der es sich um die Standard VoIP-Richtlinie handelt, die mit dem Produkt installiert wird.</span><span class="sxs-lookup"><span data-stu-id="40b64-106">Users not assigned to a voice policy will automatically be assigned to the global policy, which is the default voice policy that is installed with the product.</span></span>
