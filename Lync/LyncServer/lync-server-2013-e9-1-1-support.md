# <a name="e9-1-1-support-in-lync-server-2013"></a>Unterstützung für E9-1-1 in lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Letztes Änderungsstand des Themas:** 2012-09-21_

Lync Server 2013 unterstützt Enhanced 9-1-1 (E9-1-1) als Teil einer Unternehmensbereitstellung. Hierbei handelt es sich um eine Benachrichtigungsfunktion für Notfallsituationen, bei der die Telefonnummer des Anrufers einer allgemeinen Adresse (d. h. Straße) zugeordnet wird. Die Unterstützung für E9-1-1 steht nur in den USA zur Verfügung.

Zur Unterstützung von E9-1-1 im Rahmen einer lync Server 2013-Bereitstellung müssen Sie den E9-1-1-Routingdienst von einem zertifizierten Anbieter für Notfalldienste abrufen oder ein Elin-Gateway (Emergency Location Identification Number) verwenden. Der Anbieter von Notrufdiensten oder der Carrier leitet Notrufe, die von lync Server 2013 stammen, an den richtigen Anrufbeantworter (Public Safety Answering Points, Rettungsleitstelle), basierend auf den im Anruf enthaltenen Standortinformationen. Ausführliche Informationen zur Unterstützung von E9-1-1 finden Sie unter [Planning for Emergency Services (E9-1-1) in lync Server 2013](lync-server-2013-planning-for-emergency-services-e9-1-1.md) in der Planungsdokumentation.
